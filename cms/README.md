# Craft CMS

Headless CMS for content management. Craft provides the control panel where editors create and manage content, served to the Next.js frontend via GraphQL.

## Setup

```bash
cd cms

# Create local env file from example env
cp .env.example.dev .env

# Start DDEV + install PHP dependencies (DDEV will also
# generate TLS certs in `cms/.ddev/traefik/certs/`)
ddev composer install

# Install Craft + generate application id and security key
# (Set admin account, site name, etc. Leave Site URL blank
# if `PRIMARY_SITE_URL` is set in .env)
ddev craft install

# Open Craft control panel in browser
ddev launch
```

## License Key

Craft CMS generates a `license.key` file in `cms/config/` that pairs a website to its purchased Craft CMS and plugin licenses. This file is not committed to the Firestarter repo.

**If you are working on Firestarter itself**, add the license key to your local git exclude so you don't accidentally commit it:

```bash
echo "cms/config/license.key" >> .git/info/exclude
```

**If you are starting a new client project** from a Firestarter clone, `license.key` will be generated by Craft and should be committed to the new project repository so that licenses stay paired to the project.

> **Note:** During initial setup, Craft may generate a second key file (`license.key.1`) when you first log into the control panel. Since a key already exists from install, Craft saves the second one to a numbered file rather than overwriting. The `.1` file is safe to delete — Craft only uses `license.key`.
